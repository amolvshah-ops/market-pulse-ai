Architecture

Refined system structure

Your original prompt is strong, but for a first GitHub project it should be positioned as an execution-ready platform blueprint, not a promise of instant live profitability.

Layer 01

Research & signal agents

Technical, fundamental, sentiment, and ML agents generate normalized scores instead of free-form opinions, which makes orchestration easier to audit and backtest.

EMA, RSI, MACD, Bollinger Bands, VWAP
ROE, debt/equity, EPS growth, valuation checks
FinBERT-style news classification
Layer 02

Decision & controls

The strategy orchestrator should consume structured outputs, enforce thresholds, and route every trade through a dedicated risk gate before execution.

Weighted scoring engine
Confidence threshold + volatility filter
Session stop rules and drawdown controls
Layer 03

Execution & monitoring

Paper trading comes first. Real broker integration should remain optional until logs, backtests, and controls prove stable behavior across multiple market regimes.

Paper engine for intraday and options flows
P&L, win rate, drawdown, exposure dashboard
Broker adapter reserved for later phase

Risk framework

What makes the idea stronger

The most important edit is positioning safety, explainability, and staged rollout ahead of aggressive return claims.

Capital rules

Start with simulated capital bands, position sizing by account risk, and trade denial when loss limits are breached.

Max 2% risk per trade
Max 5% daily loss
Stop after 3 consecutive losses

Model discipline

Backtesting, walk-forward validation, and optimization should be isolated from live execution so overfitting does not leak into production decisions.

1–2 years historical data
Grid search with drawdown filters
Sharpe + win rate + stability metrics

Execution policy

Only allow execution when the orchestrator score, ML confidence, and risk context all pass; otherwise surface a clear HOLD state.

Threshold-aware entries
Volatility-aware trade reduction
Kill switch at 10% drawdown
Backend

FastAPI + task services

API routes, orchestration endpoints, trade logs, and scheduler hooks can live in a Python-first stack that is easy to run locally.

Agents

LangGraph-style coordination

A graph-based workflow fits modular agent handoffs and explicit state transitions better than loosely connected prompt chains.
